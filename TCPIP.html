TCP

Networks are number systems, this means that application layer protocols are not network friendly. TCP is used to identify application layer protocols with numbers. TCP stands for Transport Control Protocol. TCP port numbers are network addresses for service protocols. When a request is sent to a server for a specific port number of a service the server, if it is able to provide the service, will respond with the data As there are tons of application layer protocols there are tons of port numbers for these. The most common port numbers are:
80 - http
443 - https
110 - pop3
995 - secure pop3
25 - smtp
465 - secure smtp
143 - imap
993 - secure imap

TCP/IP

Each packet of data that is sent contains the IP address and the TCP number that identifies which application sends it and expects a reply.

When the reply comes the system will check the addresses and port numbers and pass the data to the right application.

TCP port numbers are randomly allocated, this allows a little bit of security and also allows a variety of applications connected to servers.

The random numbers range from 49152 to 65535 and there are 16,383 to choose from
