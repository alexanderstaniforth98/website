Inclusion

There are many problems you can have whilst creating a website in regards to the text.

One problem could be that the text is too small to read for people with visual impairments, this means the text has to be at a size which is easy for people to read. Of course the size can not be too big otherwise it would look to unprofessional, so the website has to use a size which makes it easy for every to read. I choose the size 16 pixels for my font - this makes it easy to read

Another problem that people could have with font is the fact that the font type may not be easily readable for dysleix people, they may strugle with font styles such as comic sans this is why the website is made with the font style that is easy for every one to read throughout.

Headings are used throughout my page and this is beacause I want the user to be able to quickly reach there desired information quickly. Everything on the webpage is only a couple of clicks away. Navigation bars are also used throughout the site to help users around the website

WCAG

WCAG stands for Web Content Accessibilty Guidlines and it is to make the site more userable.

Perceivable

  • This webiste contains text alterenatives for none text alterenatives
  • This website provides captions and other alternatives for multimedia.
  • Websites should create content that can be presented in different ways
  • This website makes it easier for users to see and hear content.


Operable

  • All functionality should be aavailble from a keyboard
  • Give suers enough time to read and use content
  • Do not use content that may cause seizurs
  • Help users navigate and find content.


Understandable

  • Make text readable and understandable.
  • Make content appear and operate in predictable ways.
  • Help users avoid and correct mistakes.


Robust

  • Maximize compatibility with current and future user tools.


This can be seen at https://www.w3.org/WAI/WCAG20/glance/
